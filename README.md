# 🎙️ 声チェッカー (meeting-voice-check)

オンライン会議で「自分の声が相手にどう聞こえているか」をブラウザだけで録音・再生し、定量診断する単一HTMLアプリです。サーバー不要・音声は外部送信せず、すべて端末内（ブラウザ）で処理します。

## 公開URL

GitHub Pages: https://snaka.github.io/meeting-voice-check/

## 機能

- **録音・再生** — Web Audio API で生のPCMを取得。再生音は録音波形と完全一致（WAVダウンロードも可）。「目標音量に補正して再生」で適正レベルとの聞き比べが可能。「お手本」は日本語の発話サンプル（夏目漱石『吾輩は猫である』冒頭。同梱の `ref-voice.mp3` を `decodeAudioData` で読み込み、-20dBFSに正規化）で、波形・音の両方を自分の声と比較できる。
- **3ステップのウィザードUI** — ① 録音 → ② 診断結果 → ③ シェア。結果はスコアを先頭に表示し、点数の内訳・波形は折りたたみで必要時に展開。
- **5軸の定量診断**（各100点＋総合点）
  - **音量レベル** — 発話区間のアクティブレベル(dBFS, エネルギー平均)を理想帯（-27〜-16 dBFS）と比較
  - **ノイズ・SNR** — 無音区間のノイズフロアと発話レベルの差を算出
  - **明瞭度（こもり）** — FFTで 2-4kHz(子音帯) と 0.3-1kHz(声の芯) のスペクトルバランスを評価
  - **音量の安定性** — 0.4秒窓ごとの発話レベルのばらつき（標準偏差）を評価
  - **音割れ・クリッピング** — クリップしたサンプル比率＋ピーク張り付きを検出
  - dBFSスケール上での自分の位置（平均／ピーク／ノイズ）を可視化（ⓘで単位の説明付き）
- **フィードバック（エコー）チェック** — テスト音をスピーカーから流し、マイクがその音を拾うか（＝会議で相手にエコーを返す原因になるか）を自己診断。エコーキャンセルをオフにしてスピーカー→マイクの物理的な回り込みを測定し、ベースラインに対するテスト音周波数の上昇量で判定。ヘッドホン使用なら「拾わない＝正常」。
- **結果シェア** — スコアをステッカー画像化してクリップボードにコピー／保存／Web Share。X・Bluesky はテキスト自動入力の投稿画面を起動、mixi2・Slack はコピー＆貼り付け方式。高得点ほど絵柄が非線形に増えて豪華になる演出付き。

正確に診断するため、初期状態ではブラウザの音声処理（エコー除去 / ノイズ抑制 / AGC）をオフにして生のマイク信号を測定します。会議アプリと同条件で試すトグルも用意しています。

## ローカルでの動作確認

```sh
# そのままブラウザで開く（file:// でマイクが拒否される場合あり）
open index.html

# マイクが反応しない場合はローカルサーバー経由（localhost は安全なオリジン扱い）
python3 -m http.server
# → http://localhost:8000 を開く
```

## 技術構成

- `index.html`（ビルド不要・依存なし）＋ お手本音声 `ref-voice.mp3`（約42KB）
- Web Audio API（getUserMedia / ScriptProcessor / AudioBuffer / FFT）
- 録音データ・診断結果は外部送信せず、ブラウザのメモリ内のみで処理（永続保存なし）
- アクセス解析として Google Analytics (GA4) を利用（測定ID `G-7S9C579VP5`）。音声・診断結果は送信せず、ページ閲覧などの一般的な統計のみ

## 将来の拡張（現状スコープ外）

匿名のスコア統計をバックエンド（API＋DB）で集計・比較する機能を追加する余地があります（現状は完全にクライアント完結）。
